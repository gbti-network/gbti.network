This is the exact skill I use to author every SOW in this repository. It captures the discipline that keeps our planning docs consistent: improve an existing SOW before creating a new one, ground each SOW in a real code audit so it cites file and line rather than guesses, reference the completed work it builds on, default new work to the queue lane, and follow our writing and plan-mode conventions. Point any agent that authors GBTI SOWs at it, and the SOWs come out uniform and grounded.
