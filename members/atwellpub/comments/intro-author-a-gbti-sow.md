This is the skill I reach for whenever an agent needs to author or manage a Statement of Work. It gives Claude Code a repeatable workflow: a /sow init command that scaffolds the lane folders (queue, in-progress, waiting-review, completed), then a short discipline for every SOW after that. Improve an existing SOW before creating a new one, ground each SOW in a real code audit so it cites file and line rather than guesses, reference the completed work it builds on, default new work to the queue lane, and keep its status in sync as it moves across the lanes. It is project-agnostic, so point any agent at it and your planning docs come out uniform and grounded.
